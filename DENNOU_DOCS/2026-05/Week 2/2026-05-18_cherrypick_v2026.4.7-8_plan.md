## Cherry-Pick 手順

### 基本ルール
1. 1コミットずつ `git cherry-pick <hash>` を実行
2. 競合が発生した場合は DennouAibou の修正を保持する方針で手動解決
   - watchdog timer（`state.watchdogTimer`, `startWatchdog()`）は絶対に残す
   - それ以外の上流変更は上流のコードを優先
3. 各 cherry-pick 後に @code-reviewer が検証
4. 全件完了後、Dennou テストスイートをフル実行
5. コミット＆プッシュ

### 適用順序
```
Phase 1: Heartbeat batch
  1. caecd3c1fe
  2. 2d481c9329  ← 競合リスク高。手動解決＋watchdog確認
  3. 51633fc13a
  4. aa6f6135db
  → 一括検証: heartbeatテスト＋watchdogログ確認

Phase 2: SSE batch（時系列順に厳守）
  5. d519f39c6e  ← 最古
  6. c7c0550dc9
  7. aaf5307638  ← 最新
  → 一括検証: WebUI ログ表示確認

Phase 3: 独立fix（任意順）
  8. f6bf8c7202
  9. eede8f945f
  10. 681931345b
  11. 7306cf3707

Phase 4: pi-embedded-runner batch
  12. adb7b0d5d6  ← DennouAibou修正維持
  13. 594ea6e1b9  ← 上流そのまま取る
```
